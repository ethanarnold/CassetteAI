# CassetteAI — Unified Spec

> Single source of truth: architecture, detailed component specs, and build progress.

## One-Liner

Natural language → Claude → generative DNA model → tissue-specificity scoring → Claude interpretation → cassette design.

## Problem

AAV gene therapies need tissue-specific regulatory elements (promoters/enhancers) to avoid off-target expression that causes toxicity and immune responses. Designing these elements currently requires months of iterative wet-lab screening across large combinatorial libraries. No tool lets a gene therapy developer describe what they need in plain English and get a scored, ranked set of candidates back in minutes.

## What We're Building

A pipeline where a user types something like:

> "Design a liver-specific enhancer for AAV delivery, under 500 base pairs"

And gets back:

1. Candidate 200 bp regulatory elements generated by a conditional diffusion model
2. Tissue-specificity scores across dozens of cell/tissue types
3. Claude's biological interpretation: ranking, rationale, pathology flags
4. A final cassette diagram showing the element composed with a minimal promoter

### 1A. DNA-Diffusion Generator (`backend/modal_generate.py`)

**What it is:** Conditional denoising diffusion model that generates 200 bp synthetic regulatory DNA elements conditioned on cell type.

**Repository:** https://github.com/pinellolab/DNA-Diffusion

**Key details:**
- Architecture: U-Net with cell-type conditioning via class labels
- Input: cell-type label (e.g., "HepG2") + Gaussian noise matrix (4 × 200, one-hot encoded)
- Output: 200 bp DNA sequence
- Pretrained on: DNase I hypersensitive sites from ENCODE (K562, HepG2, GM12878)
- Inference: ~50 denoising steps per sequence
- Generation of 200 sequences: ~1-2 min on A100

### 1B. Sei Scorer (`backend/modal_score.py`)

**What it is:** Deep CNN that predicts 21,907 chromatin profiles (TF binding, histone marks, DNase accessibility) from a 4,096 bp DNA sequence input.

**Repository:** https://github.com/FunctionLab/sei-framework
**HuggingFace:** https://huggingface.co/spaces/FunctionLab/sei

**Key details:**
- Input: 4,096 bp DNA sequence (center your 200 bp element in a neutral flanking context)
- Output: 21,907 predictions grouped into 40 "sequence classes" (regulatory activity profiles)
- Relevant sequence classes for liver specificity: "Liver / HepG2" enhancer and promoter classes
- Inference: ~0.5-1 sec per sequence on GPU
- Scoring 200 sequences: ~2-5 min on A100

### 1C. Cache Layer (`backend/cache.py`)

**Purpose:** Ensure demo reliability. Serve pre-computed results for known prompts. Fall through to live compute for novel prompts.

```python
import hashlib
import json
import os

CACHE_DIR = "pipeline_cache"

def cache_key(prompt: str) -> str:
    normalized = prompt.lower().strip()
    # Map common phrasings to canonical forms
    for tissue in ["liver", "hepatocyte", "hepatic"]:
        if tissue in normalized:
            normalized = "liver_specific_enhancer"
            break
    for tissue in ["heart", "cardiac", "cardiomyocyte"]:
        if tissue in normalized:
            normalized = "cardiac_specific_enhancer"
            break
    for tissue in ["brain", "neuron", "neural", "cns"]:
        if tissue in normalized:
            normalized = "neural_specific_enhancer"
            break
    return hashlib.sha256(normalized.encode()).hexdigest()[:16]

def get_cached(prompt: str, stage: str):
    key = cache_key(prompt)
    path = os.path.join(CACHE_DIR, key, f"{stage}.json")
    if os.path.exists(path):
        with open(path) as f:
            return json.load(f)
    return None

def set_cache(prompt: str, stage: str, data):
    key = cache_key(prompt)
    os.makedirs(os.path.join(CACHE_DIR, key), exist_ok=True)
    path = os.path.join(CACHE_DIR, key, f"{stage}.json")
    with open(path, "w") as f:
        json.dump(data, f)
```

**Pre-compute for these prompts:**
1. `"Design a liver-specific enhancer for AAV delivery"` — primary demo
2. `"Design a cardiac-specific enhancer for AAV delivery"` — encore/Q&A
3. `"Design a neuron-specific enhancer for AAV delivery"` — backup

Each cached run stores: `generation.json`, `scoring.json`, `interpretation.json`, `cassette.json`.

**Build checklist:**

- [ ] Implement `get_cached(prompt, stage)` and `set_cache(prompt, stage, data)`
- [ ] Cache directory structure: `cache/{key}/{stage}.json`

**Post-scoring interpretation rules:**
- Rank candidates by on-target Sei signal

- Recommend a final candidate and compose it with a minimal promoter
- Output a natural language summary suitable for a non-computational biologist

## Biological Glossary

- **AAV (Adeno-Associated Virus):** Viral vector used to deliver gene therapies. Packaging limit ~4.7kb of DNA.
- **Promoter:** DNA sequence that initiates transcription of a gene. Determines where/when/how much a gene is expressed.
- **Enhancer:** DNA sequence (often 200-1000bp) that boosts transcription from a distant promoter. Can be tissue-specific.
- **Minimal promoter (minP):** A stripped-down promoter (e.g., minTBG, minCMV) that has basal transcription machinery binding but no tissue specificity on its own. Needs an enhancer to drive strong/specific expression.
- **Tissue specificity:** The property of a regulatory element driving expression only in the target tissue and staying silent everywhere else.
- **HepG2:** Human hepatocellular carcinoma cell line. Standard proxy for liver/hepatocyte gene regulation.
- **ENCODE:** Encyclopedia of DNA Elements. Massive consortium dataset of regulatory element annotations across cell types.
- **Sei:** A deep learning model that predicts 21,907 chromatin/regulatory profiles from DNA sequence.
- **Cassette:** The complete DNA payload inside an AAV: ITR—enhancer—promoter—transgene—polyA—ITR.
- **ITR (Inverted Terminal Repeat):** Hairpin structures at each end of AAV genome required for packaging. ~145bp each.
- **polyA signal:** Sequence (typically from bGH or SV40) that terminates transcription. ~200-250bp.
